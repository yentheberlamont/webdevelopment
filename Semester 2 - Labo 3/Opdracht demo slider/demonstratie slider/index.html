2.1.1) Waar wordt de event listener gekoppeld aan de slider?

  • Antwoord: De eventlistener wordt gekoppeld aan de slider in de setup-functie, in de volgende regels code:
  • sliders[0].addEventListener("change", update);
  • sliders[0].addEventListener("input", update);

2.1.2) Waarom moeten we die op twee soorten events koppelen?

  • Antwoord: Het ene event is voor als de slider verplaatst wordt, terwijl de andere is voor wanneer de slider losgelaten wordt

2.1.3) In de CSS file wordt nergens een rode kleur opgegeven, waar wordt dan wel de rode kleur van het blokje ingesteld?

  • Antwoord: De kleur van het blokje wordt ingesteld in de javascript code in de volgende code:
  • colorDemos[0].style.backgroundColor="red";

2.1.4) Waarom schrijven we telkens sliders[0] en colorDemos[0] en niet gewoon sliders of colorDemos?

  • Antwoord: Er is maar 1 html-element met uit de klasse sliders, en 1 uit de klasse colorDemos. In javascript wordt er toch nog steeds een array geretourneerd, maar er zit maar 1 element in. Door die [0] er bij te zetten, gaan we alsnog dus die specifieke elementen targetten.

2.2) Waar vind je de CSS-properties die programmatorisch werden ingesteld, zoals bv. de rode achtergrondkleur van het blokje?

  • Antwoord: Het blokje krijgt een kleur in de html-code, en voor de kleur te veranderen wordt er gewerkt in de javascript code.
